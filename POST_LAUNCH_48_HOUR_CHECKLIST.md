# BharathQR Post-Launch 48-Hour Checklist

## First 2 hours

- Homepage opens.
- Tool pages open.
- Solutions/use-case/material pages open.
- Sitemap opens.
- robots.txt opens.
- No obvious mobile layout break.

## First 24 hours

- Submit sitemap in Google Search Console.
- Submit sitemap in Bing Webmaster Tools if available.
- Run SEO agent workflow manually once.
- Confirm GitHub Actions does not fail.
- Check Vercel latest production commit.

## First 48 hours

- Add 3–5 founder notes in `feedback/inbox.md` if content feels off.
- Check Search Console coverage status.
- Check generated blog quality.
- Do not add major new features until route stability is confirmed.
