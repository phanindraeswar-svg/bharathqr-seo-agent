# Wave 20 — Founder Growth Ops Summary

This wave adds a lightweight owner-review layer so BharathQR can keep improving with minimal manual work after deployment.

## Added

- `data/founder_review_queue.json`
- `scripts/founder_review_digest.py`
- `scripts/weekly_growth_planner.py`
- `FOUNDER_REVIEW_DIGEST.md` generated by script
- `WEEKLY_GROWTH_PLAN.md` generated by script

## New commands

```bash
npm run founder-digest
npm run weekly-growth-plan
npm run owner-review
```

## Purpose

Instead of asking the founder to inspect hundreds of pages, the system creates a small review digest and a weekly growth plan.

Founder only needs to review:

1. Latest live routes/tools.
2. Latest generated blog style.
3. Top opportunities.
4. Search Console issues after data exists.

This fits the project rule: minimal manual work, maximum automation.
