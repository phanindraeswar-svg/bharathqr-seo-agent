# BharathQR Post-Launch Operating System

## Daily — 2 minutes
- Check Vercel deployment status.
- Check GitHub Actions status.
- Add a founder note only if something feels wrong.

## Weekly — 10 minutes
- Run `npm run post-launch-report`.
- Review Google Search Console top pages and queries.
- Improve 1–3 pages with impressions but low CTR.
- Do not add random tools unless data supports it.

## Monthly — 30 minutes
- Refresh weak content.
- Expand only clusters that receive impressions.
- Review monetization readiness.

## Rule
After launch, Search Console data beats guesses. Founder feedback still beats generic AI writing.
